### 6.4 Pass-1-replacement comparison under matched Pass 2

The final results question is the paper's narrow external-value claim: whether the trajectory produced by the proposed framework is useful as the initialization stage of a downstream direct-collocation pipeline on the same problem setup. We frame the comparison as a pipeline-variant replacement rather than a naive-versus-warm-started contrast. Specifically, we compare the full two-pass direct-collocation method (DCM) — in which a Hermite-Simpson Pass 1 produces the initial thrust profile and phase structure that seed a multi-phase Legendre-Gauss-Lobatto Pass 2 — against a variant in which the Bézier successive-convexification optimizer replaces Pass 1 while Pass 2, the dynamics model, the path and boundary constraints, the solver implementation, the tolerances, the stopping rules, and the free-true-anomaly boundary-condition protocol all remain identical. The only difference between the two pipelines is the origin of the warm-start trajectory and the phase-structure decision fed into Pass 2. This framing isolates the replacement question and avoids implying that the proposed method competes against direct collocation as a method class.

The naive-baseline stage against which the Bézier upstream is measured is therefore the full Hermite-Simpson Pass 1 as implemented in the two-pass DCM pipeline: a uniform-mesh collocation on the same dynamics model, with the same thrust representation and the same peak-detection post-processing that determines phase boundaries for Pass 2. The Bézier warm-start export mapping samples the optimized Bézier curve and its reconstructed thrust profile `u(τ) = r''(τ)/T² − g(r(τ))` at the same number of nodes used by Pass 1, passes the resulting `(t, x, u)` triples through the same `detect_peaks` and `determine_phase_structure` routines that Pass 1's output is passed through, and hands the resulting warm-start state and phase structure directly to Pass 2 via the same multi-phase LGL interpolation used by the baseline. Pass 2 receives only a state and control trajectory guess and a phase-boundary decision; no additional collocation variables are initialized from the Bézier run, because Pass 2 regenerates its own LGL node variables internally.

Under this matched protocol, the primary reported metrics are solve success, end-to-end solve time with the runtime decomposed into the stages actually incurred by each pipeline, final objective value, and the peak count recovered by the phase-structure decision. The comparison is scoped to the set of circular-to-circular transfers in which both pipelines converge. Two independent boundaries define this scope. First, the Bézier upstream is feasible only on circular orbits in the tested dataset: across a 10-case eccentricity run with `T_normed ≤ 0.5` and `max(e0, ef) ≤ 0.1`, the six mildly elliptic cases all had infeasible Bézier upstreams, while the four circular cases all succeeded; the 112-case boundary sweep corroborates this at larger scale, with `0` eccentric cases having a feasible Bézier upstream. Second, transfer time constrains downstream convergence: at `T_normed > 1.0`, the Bézier upstream is often still feasible (`27 / 105` cases) but Pass 2 rarely converges from the resulting warm-start (`2 / 105` both-converge). The scoped regime is therefore a minority of the trajectory database.

End-to-end speedup is `>1×` in four of the seven cases (cases 2, 6, 39, 126) and `<1×` in three (cases 4, 20, 114). The median speedup is `1.22×`, with a range of `0.51×` to `2.47×`. Case 20 — a `1.92`-revolution transfer — incurs a markedly longer Pass 2 solve from the Bézier warm-start (`6.60 s`) than from the Pass 1 warm-start (baseline `3.51 s` total), indicating that the Bézier upstream's warm-start is not always better-aligned with Pass 2's optimum than Pass 1's. The regime-stratified feasibility and convergence counts are documented in `doc/dcm_downstream_pack.md`.

Two interpretation boundaries must remain explicit. First, Table 6 supports only a pipeline-variant initialization claim under the matched Pass 2 protocol and within the scoped regime (circular orbits with both pipelines converging). It does not show that the proposed method is better than direct collocation or that it should replace downstream solvers; the proposed pipeline *is* a DCM pipeline, differing from the baseline only in how Pass 1 is carried out. Second, the small sample (seven both-converged cases) and the narrowness of the scoped regime mean that the numerical speedup distribution should not be extrapolated to the population of cases where Pass 2 fails from the Bézier warm-start. The honest reading is that the Bézier upstream is a usable Pass 1 replacement under the stated assumptions, preserving final cost on `6 / 7` tested cases and reducing end-to-end runtime on `4 / 7`, not that it is a general downstream accelerator.
